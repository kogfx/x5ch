# x5ch - Terminal 5ch Browser & Discord Archiver

**x5ch** は、ターミナル（CLI）で動作する軽量かつ高速な5ちゃんねるブラウザです。
キーボード操作のみで効率的に閲覧できるほか、気に入ったスレッドや未読レスをDiscordサーバーへ自動転送・アーカイブする機能を備えています。

## ✨ 主な機能

* **ターミナル完結**: マウス不要。キーボード操作のみで完結する高速ブラウジング。
* **テキスト特化**: 画像の読み込み待ちや広告に邪魔されず、テキストのみを快適に閲覧可能。
* **Discord連携 (画像自動展開)**:
    * 未読レスをDiscordのフォーラム/チャンネルへ自動転送。
    * スレッドごとにDiscord側のスレッドを自動作成。
    * 2000文字を超える長文レスは自動分割して送信。
    * **h抜きURL (ttps://) の自動補完**: ターミナルではテキストのみ、Discord側では画像プレビューという使い分けが可能。
* **強力な未読管理 & Cron連携**:
    * 閲覧履歴に基づき、バックグラウンドで新着レスを追跡・転送。
    * PCに触れていない間も自動でアーカイブを作成します。
* **スマートな排他制御**:
    * 自動巡回(cron)中に手動ブラウザを起動すると、自動的にバックグラウンド処理を中断・引き継ぎを行い、スムーズに操作を開始できます。
* **レジューム機能**:
    * 履歴機能により、前回閲覧した続きからスムーズに再開・転送が可能です。

## 📦 動作環境

* Ruby 3.0 以上推奨
* 標準ライブラリ (`net/http`, `json`, `uri` 等) のみで動作するため、`bundle install` 等の複雑な作業は不要です。

## 🚀 インストールとセットアップ

### 1. リポジトリのクローン

```bash
git clone https://github.com/kogfx/x5ch.git
cd x5ch
```

### 2. 設定ファイルの作成

サンプルの設定ファイルをコピーして、本番用の設定ファイルを作成します。
**注意**: `config.rb` は Git の管理対象外（.gitignore）になっています。

```bash
cp config.sample.rb config.rb
```

### 3. トークンの設定

作成した `config.rb` をエディタで開き、Discord Bot のトークンと、転送先のチャンネルIDを入力してください。
（設定値の取得方法は、後述の「補足: Discord設定の手順」を参照してください）

```ruby
module Config
  # --- Discord設定 ---
  # Botのトークンと、転送先のチャンネルIDを設定してください
  DISCORD_BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"
  DISCORD_CHANNEL_ID = "YOUR_CHANNEL_ID_HERE"

  # --- 保存ファイルパス (通常は変更不要) ---
  HISTORY_FILE = File.join(Dir.home, ".x5ch_history.json")
  QUEUE_FILE   = File.join(Dir.home, ".x5ch_queue.json")
  LOCK_FILE    = File.join(Dir.home, ".x5ch.lock")
  PID_FILE     = File.join(Dir.home, ".x5ch.pid")
end
```

## 📖 使い方: ブラウザモード (`x5ch.rb`)

普段のブラウジングに使用します。

```bash
./x5ch.rb
# または
ruby x5ch.rb
```

### キー操作一覧

| キー | 動作 | 対象画面 |
| :--- | :--- | :--- |
| **数字** | 項目を選択して移動 / 決定 | 全般 |
| **Enter** | 次のページへ進む | 全般 |
| **p** | 前のページへ戻る (Previous) | 全般 |
| **b** | 階層を一つ戻る (Back) | 全般 |
| **q** | アプリを終了 / 一つ戻る | 全般 |
| **s** | キーワード検索 / 絞り込み | 全般 |
| **r** | 最新情報に更新 (Reload) | 板/スレッド一覧 |
| **m** | **【重要】** 選択したスレを転送キューに追加 | スレッド一覧 |
| **t** | 転送待ちキューの確認・削除 (Task) | 全般 |
| **H** | 履歴の管理・削除 (History) | メイン/スレッド一覧 |
| **Ctrl+C** | 転送キューを保存して強制終了 | 全般 |

### 画面の見方
* **水色文字**: 「最近読んだスレッド」または「転送キューに入っているスレッド」。
* **`✔` マーク**: 既読（最終確認時から新着がない状態）。
* **`+` マーク (赤)**: 未読（新着レスがある状態）。

## 🤖 使い方: 自動巡回モード (`cron_x5ch.rb`)

画面を表示せず、バックグラウンドで「履歴にあるスレッド」の新着チェックと転送を行います。

```bash
./cron_x5ch.rb
# または
ruby cron_x5ch.rb
```

### 動作仕様
1.  **未読チェック**: 履歴にあるスレッドを巡回し、新着レスがあるか確認します。
2.  **転送**: 新着があれば転送キューに追加し、Discordへ送信します。
3.  **排他制御**:
    * 手動ブラウザ (`x5ch.rb`) が起動中の場合、**何もせずに即終了**します。
    * 自動巡回中に手動ブラウザが起動された場合、**処理を中断・保存して終了**し、手動操作へ即座に譲ります。

### Cronへの登録例

例：30分おきに自動巡回させる設定。

```bash
# crontab -e で編集
*/30 * * * * cd /path/to/x5ch && /usr/bin/ruby cron_x5ch.rb >> cron.log 2>&1
```

## 🔧 補足: Discord設定の手順

Discord連携に必要な「Botトークン」と「チャンネルID」の取得手順です。

### 1. 専用サーバーの作成
1. Discordアプリ左側のサーバーリスト最下部にある「＋」ボタンを押します。
2. 「オリジナルの作成」→「自分と友達のため」などを選び、適当な名前（例: `5ch-Log`）をつけてサーバーを作成します。

### 2. Botの作成とトークンの取得
1. ブラウザで [Discord Developer Portal](https://discord.com/developers/applications) にアクセスし、ログインします。
2. 右上の **[New Application]** をクリックし、名前（例: `x5ch-bot`）を入力して作成します。
3. 左メニューの **[Bot]** を選びます。
4. **[Reset Token]** ボタンを押し、表示されたトークンをコピーします。
   * これが `config.rb` の **`DISCORD_BOT_TOKEN`** です。

### 3. Botをサーバーに招待する
1. 左メニューの **[OAuth2]** → **https://www.homedepot.com/b/Outdoors-Outdoor-Power-Equipment-Generators/N-5yc1vZbx8l** を選びます。
2. **SCOPES** の項目で `bot` にチェックを入れます。
3. 下に現れる **BOT PERMISSIONS** で以下にチェックを入れます。
   * `Send Messages`
   * `Read Messages/View Channels`
   * `Embed Links`
   * `Attach Files`
4. 一番下に生成されたURLをコピーし、ブラウザのアドレスバーに貼り付けて移動します。
5. 作成したサーバーを選択して「認証」すると、Botがサーバーに参加します。

### 4. チャンネルIDの取得
1. Discordアプリの左下にある歯車アイコン（ユーザー設定）を開きます。
2. 左メニューの **[詳細設定 (Advanced)]** を選び、**[開発者モード (Developer Mode)]** をONにします。
3. 設定を閉じ、転送先にしたいチャンネル（例: `#general`）の名前を**右クリック**します。
4. **[チャンネルIDをコピー]** をクリックします。
   * これが `config.rb` の **`DISCORD_CHANNEL_ID`** です。

## ⚠️ 注意事項・Tips

* **Ctrl+C の挙動**: 
  転送処理中に `Ctrl+C` を押しても、処理中のタスクはキューの先頭に戻されて保存されます。次回起動時に続きから再開されるため、データロストの心配はありません。

* **ファイル構成**:
  履歴データ(`~/.x5ch_history.json`)やキュー(`~/.x5ch_queue.json`)はホームディレクトリに保存されます。

* **5chへの負荷**:
  スクレピングを行うため、過度なリロードや短時間での大量転送は規制の対象となる可能性があります。常識の範囲内でご利用ください。
